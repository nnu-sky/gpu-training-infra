# GPU 训练性能优化

面向 Codex 的 PyTorch 与 CUDA 训练工程优化技能。它通过可比较的基线和短基准定位主要瓶颈，再验证少量具有明确依据的候选，避免把最高瞬时吞吐直接当作最佳训练配置。

## 能做什么

- 分析 GPU 利用率、吞吐、显存、数据等待和完整训练墙钟时间；
- 比较 FP32、BF16、FP16、`channels_last` 和 `torch.compile`；
- 评估批量拐点、更新次数、学习率联动风险和预计训练成本；
- 定位数据加载、尾批动态形状、同步、日志、评估和检查点开销；
- 评估 DDP、FSDP 和多卡扩展效率；
- 输出一套有测量依据的推荐组合，并区分单项、增量和组合收益。

它不会把某个项目的最佳参数当成跨项目默认值，也不会为了性能分析自动创建项目管理文档。

## 项目规则

技能会优先遵守项目已经存在的说明、实验协议和 `AGENTS.md`。这些文件都不是前置条件：

- 存在时读取并遵守；
- 不存在时，从项目说明、配置、训练入口和已有日志提取约束；
- 在执行修改或实验前列出必要假设；
- 不会仅为运行本技能要求用户创建 `AGENTS.md`。

## 安装

### 方式一：作为插件安装

把本仓库加入 Codex 插件市场源：

```bash
codex plugin marketplace add nnu-sky/gpu-training-infra
```

刷新插件市场后安装“GPU 训练性能优化”。这种方式适合希望后续统一升级的人。

### 方式二：只安装技能

在 Codex 中调用 `$skill-installer`，并让它安装：

```text
https://github.com/nnu-sky/gpu-training-infra/tree/main/plugins/gpu-training-infra/skills/gpu-training-infra
```

安装后若没有立即出现，重新启动 Codex。

## 使用

可以显式调用：

```text
使用 $gpu-training-infra 分析这个 PyTorch 项目的训练瓶颈，先建立基线，再给出最推荐的工程优化组合。
```

也可以直接描述训练提速、显存、数据加载、批量、混合精度、`torch.compile` 或多卡扩展问题，Codex 会根据技能描述判断是否调用。

## 安全与适用范围

- 仓库不包含服务器地址、账号、密钥、令牌、个人路径或私有项目配置；
- 优化结论必须同时报告速度、显存和项目目标指标；
- 短基准只能用于筛选，最终接受仍需使用可比较的训练窗口或完整训练验证；
- 使用脚本和修改训练代码前，应先阅读对应内容并确认适合当前项目。
