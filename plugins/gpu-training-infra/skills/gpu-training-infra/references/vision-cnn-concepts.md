# 视觉训练优化概念

仅在用户要求理解术语、训练流程或优化原理时读取。解释时先说明它优化了哪一段，再说明收益、代价和验证方法。

## 一批数据如何经过训练

1. CPU 工作进程读取、解码并增强下一批图像。
2. 主机内存中的批次传到 GPU。
3. GPU 完成前向计算、损失、反向传播和参数更新。
4. 单卡重复上述过程；多卡时各卡计算不同批次，再同步梯度。

理想状态是 CPU 准备第 `t+1` 批时，GPU 正在计算第 `t` 批，二者尽量重叠。性能优化是在不偷换模型、数据和指标的前提下，减少等待、传输、计算、同步或启动开销。

## 数据侧

- 异步数据加载：设置多个 `DataLoader` 工作进程，让读取和增强在训练主进程之外提前进行。工作进程过少会让 GPU 等数据，过多会增加进程、内存和调度开销。
- 页锁定内存：把待传输批次放在不会被操作系统换出的主机内存中，使 GPU 直接内存访问更高效。它通常与 `non_blocking=True` 一起测试；会增加主机内存压力，并非单独开启就必然提速。
- 常驻工作进程：跨训练周期保留数据工作进程，省去重启开销。它会改变工作进程随机数的生命周期，严格复现实验必须单独核对增强序列。
- 预取：每个工作进程提前准备若干批次。预取不足可能断粮，过多会占用内存并制造无用工作。

## GPU 计算侧

- 混合精度：卷积和矩阵乘等适合的算子使用 `bf16` 或 `fp16`，敏感算子保留 `fp32`。它可减少显存和提高 Tensor Core 吞吐；`fp16` 训练通常需要梯度缩放，二者都要验证损失稳定性和最终指标。
- `channels_last`：不改变图像数值，只改变四维张量在内存中的排列，使部分卷积低精度内核更高效。模型和输入必须配套转换；隐式转换或小图像任务可能抵消收益。
- cuDNN 自动调优：固定输入形状时试跑多个卷积算法并缓存最快者。开始阶段会变慢，动态形状会反复选择，而且吞吐模式可能不满足严格确定性要求。
- `torch.compile`：捕获 PyTorch 运算图并融合算子，减少 Python 和内核启动开销。首次编译、不同形状重编译、图断裂、工具链和额外显存都可能让短任务更慢，必须分开报告编译时间与稳定时间。
- 梯度置空：`zero_grad(set_to_none=True)` 不把每个梯度张量逐元素清零，而是在下次反向传播时重新创建或写入，可减少一次内存操作；要核对项目是否依赖“梯度张量始终存在”。

## 多卡

- DDP：每张 GPU 对应一个训练进程和一份模型副本，各自处理不同数据，反向传播时通过 NCCL 汇总梯度。大任务可缩短每轮时间；小模型或小批量可能被通信开销抵消。
- 固定全局批量用于测试强扩展，实验定义较接近单卡；固定每卡批量会增大全局批量，主要测试最大吞吐，属于新的训练配方。
- 扩展效率计算为：`多卡吞吐 ÷（GPU 数量 × 单卡吞吐）`。
- 能放入单卡的 ResNet34 不需要 FSDP；只有模型、激活或优化器状态无法合理放入单卡时才考虑分片。

## 一个具体例子

若 ResNet34 每批 128 张 CIFAR100 图像：8 个 CPU 工作进程准备下一批，页锁定内存和非阻塞传输把批次送入 GPU；GPU 用混合精度执行卷积，必要时配合 `channels_last`；`torch.compile` 只有在编译成本能被长训练摊薄且稳定步骤更快时保留。若再用两张卡，DDP 让两卡各处理一部分数据并同步梯度，只有同步后的总吞吐明显高于单卡才有价值。
